Geometry breakdown

From flat SVG → volumetric scene

Each SVG primitive maps to an extruded mesh with independent depth and animation channel.

  • Base plate — rounded box, 12px corner radius, subtle bevel
  • Brackets — tube geometry along bracket paths, glass-metallic stroke
  • Developing cells — extruded rounded rects, emissive pulse (z-lift)
  • Planned cells — torus rings, opacity breathe
  • Mastered cells — extruded tile + check emboss, specular highlight sweep

Animation choreography

  • Slow orbital camera + 8° idle yaw on the logo group
  • Staggered cell pulses mapped to plan status semantics
  • Rim light orbit + bloom on accent emissive (Three.js UnrealBloomPass)
  • prefers-reduced-motion → static pose, bloom only
Current — CSS 3D tilt

Flat SVG texture on a card; rotateX/Y simulates depth. No real geometry or parallax between grid cells.

Proposed — extruded WebGL

Real mesh depth: brackets stand off the plate, status tiles float at different Z, lighting creates specular edges. Reads as a crafted object, not a spinning sticker.

Drag the hero canvas to orbit · scroll to zoom · use SVG overlay to verify shape alignment

Stack (production)

Three.js r162+ as ES module (~45KB gzipped tree-shaken). Lazy-load on home route mount. Fallback to flat SVG if WebGL unavailable.

Performance target

<16ms/frame on M1 / integrated GPU. ~120 triangles. Single draw-call batch via merged geometry optional. Pause when tab hidden.

Theme sync

Material palette swaps on [data-theme] — navy plate + #5cb1ff ink (dark), white plate + #3b82f6 ink (light). Same topology both themes.
